## Project Overview

This is a **CDI (Capacitive Deionization) analytics pipeline** targeting Q1 journal publication (Desalination / Separation and Purification Technology). CDI is an electrochemical water desalination technology: saline water flows between charged electrodes, ions are adsorbed (desalination phase), then the electrodes are regenerated by reversing/removing the potential (regeneration phase). The experiment logs conductivity and current at 1 Hz throughout each cycle.

The core contribution is a **two-stage Gaussian Process (GP) framework** that replaces fixed charge-efficiency assumptions with a learned conductivity proxy **Λκ (Lambda-kappa)**. Stage 1 learns Λκ from operating conditions; Stage 2 uses Λκ as a feature to predict energy consumption and desalination performance.

**Framing**: This is a **methodology paper**, not an optimisation study. With only 10 unique operating conditions, claims must be limited to condition-level trends. The ~17–21% ablation improvement is the central quantitative claim.

## Dataset

Raw data lives in `Dataset/` (do not modify):

| File | Sheets (= varied parameter values) | Fixed conditions |
|---|---|---|
| `1000-2500-5000-7500 ppm.xlsx` | 1000, 2500, 5000, 7500 ppm | flow=3 mL/min, potential=1.8 V |
| `vary flow rate.xlsx` | 2, 3, 4, 5 mL/min | conc=1000 ppm, potential=1.8 V |
| `Vary potential-1000ppm.xlsx` | 0.9, 1.2, 1.5, 1.8 V | conc=1000 ppm, flow=3 mL/min |
| `Combined_CDI_DATA.xlsx` | Single `Data` sheet | Pre-computed cycle-level metrics |

Each raw sheet contains per-second timeseries: `Time (s)`, `Conductivity (mS/cm)`, `Current (mA)`.

`Combined_CDI_DATA.xlsx` contains cycle-level summaries: `time, conductivity, current, flow, potential, concentration, cycle_id, phase (desal/regen), time_to_max_desal, Δc, Removal%, W_net, SEC_net, ENAS`.

**10 unique operating conditions** (concentration ppm, flow mL/min, potential V):
`(1000,2,1.8), (1000,3,0.9), (1000,3,1.2), (1000,3,1.5), (1000,3,1.8), (1000,4,1.8), (1000,5,1.8), (2500,3,1.8), (5000,3,1.8), (7500,3,1.8)`

Cycle counts per condition range from 4–18 desal cycles; ~78 total desal cycles across all conditions.

## Pipeline Architecture

The pipeline is a linear sequence of stages. All outputs go to `cdi_output/`.

### Stage 0 — Data Loading & Cycle Segmentation
- Load raw timeseries from all three Excel files; annotate each sheet with its operating condition metadata (concentration, flow, potential).
- Detect desal/regen phase boundaries from current sign in the timeseries.
- **Critical**: iterate over **all cycles** in each condition — the known bug in the old pipeline was saving only the first cycle (producing ~10 instead of ~78 data points).

### Stage 1 — Cycle Metric Extraction
Compute per-cycle scalar metrics from the segmented timeseries:
- **Λκ**: `∫Δκ dt / ∫I dt` — conductivity reduction per unit charge (the proxy, **not** physical charge efficiency)
- **SEC** (Specific Energy Consumption, kWh/m³)
- **R_kappa_pos**: conductivity removal in the desalination phase
- Supporting metrics: Δc (mg/L), Removal%, W_net, ENAS

### Stage 2 — Two-Stage GP Models
**Stage 1 GP**: predicts Λκ from `(concentration, flow, potential)`.
**Stage 2 GP**: two separate GP models predicting `SEC_Wh_m3` and `R_kappa_pos` from `(concentration, flow, potential, Λκ)`.

Flow handling decision (T-03): either include flow in Stage 2 GP (Option A, preferred) or document the fixed/median flow used and justify exclusion (Option B).

### Stage 3 — LOCO-CV Validation
Leave-One-Condition-Out Cross-Validation: hold out one of the 10 conditions, train on the remaining 9, predict on **all cycles** of the held-out condition. Output: one prediction row per cycle + condition-level summary (mean ± std). Validation CSV must show n_cycles ≈ 78 (not 10).

### Stage 4 — Uncertainty Calibration
Empirical multiplier applied to GP posterior std from LOCO-CV residuals. Target: 85–95% adjusted coverage for the nominal 90% credible interval on all three models (Λκ, SEC, R_kappa_pos).

### Stage 5 — Ablation Study
Three approaches compared on both SEC and R_kappa_pos objectives:
1. Ridge regression (baseline)
2. GP without Λκ
3. GP with Λκ (full model)

Output: single table — rows = approach, columns = MAE/RMSE/Rel.MAE for each objective.

### Stage 6 — Region-Based Validation
Split predictions into near-observed vs. far-extrapolation regions; compute and compare MAE per region.

### Stage 7 — Pareto Optimisation
Pareto front between SEC (minimise) and R_kappa_pos (maximise). Identify knee candidate. The knee must report all three operating coordinates `(concentration, flow, potential)` plus predicted objectives with calibrated uncertainty bands.

**Language rules** — never use these phrases:
- ~~optimal solution / optimal operating point~~ → "model-suggested candidate for experimental validation"
- ~~maximum removal~~ → "highest predicted conductivity removal in near-observed region"
- ~~true efficiency~~ → "conductivity-based efficiency proxy (Λκ)"
- ~~proven global optimum~~ → "Pareto knee candidate — subject to experimental confirmation"
- ~~charge efficiency~~ → "conductivity-native efficiency proxy"

### Stage 8 — Publication Figures (7 total, ≥300 DPI)
1. Pipeline schematic: raw data → segmentation → cycle metrics → Stage 1 → Stage 2 → Pareto
2. Cycle segmentation example: current + conductivity vs. time, phase boundaries annotated
3. Conductivity removal profile: κ(t) for a representative cycle with R_kappa area shaded
4. Λκ sensitivity surface: Λκ vs. concentration and potential (flow at median)

## Key Scientific Constraint — Λκ Definition

Λκ is **not** physical charge efficiency. It is a conductivity-native proxy: conductivity reduction per unit charge derived purely from available sensors. It is valid as a relative comparator within the same system; it cannot be converted to absolute moles removed without Cout calibration.
